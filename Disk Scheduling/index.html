Overview of Disk Scheduling Algorithms

FCFS

Serves requests in arrival order. Simple but may cause high seek latency with no optimization.

SSTF

Selects the closest request to the current head position. Reduces seek time but may cause starvation.

RDS

Picks requests randomly without any order logic. Unpredictable and not optimal — used as a baseline comparison.

PDS

Schedules requests based on user-assigned priority. Higher priority requests served first. May starve low-priority ones.

SCAN

Moves head in one direction servicing requests, then reverses. Elevator-like behavior.

C-SCAN

Moves in one direction, jumps to start after reaching end. Uniform wait times.

LOOK

Like SCAN but stops at last request instead of disk end. More efficient movement.

C-LOOK

Circular version of LOOK. Jumps directly to the first request after reaching last one.
